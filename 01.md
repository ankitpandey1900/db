```sql
CREATE VIEW student_marks_all AS
SELECT s.student_id, s.name, sub.subject_name, m.exam_name, m.marks
FROM students s

JOIN subject sub ON m.subject_id = sub.subject_id;
```

## 🔹 1) Function: Calculate Absent Days

### ✅ SQL

```sql
DELIMITER //

CREATE FUNCTION get_absent_days(
    total_days INT,
    present_days INT
)
```

## 🔹 7) Cursor: Display names one by one

```sql
DELIMITER //

CREATE PROCEDURE display_students()
BEGIN
    DECLARE done INT DEFAULT FALSE;
    DECLARE s_name VARCHAR(50);

    DECLARE cur CURSOR FOR SELECT name FROM students;
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = TRUE;

    OPEN cur;

    read_loop: LOOP
        FETCH cur INTO s_name;
        IF done THEN
            LEAVE read_loop;
        END IF;
        SELECT s_name;
    END LOOP;

    CLOSE cur;
END //

DELIMITER ;
```

---

### 🧠 Why

* Cursor = row by row processing

### 🧠 HinEnglish

"Loop jaisa behavior SQL me."

## 🔹 5) Trigger: Track Class Change

### ⚠️ Important concept

Real-world audit logging

---

### ✅ First create history table

```sql
CREATE TABLE student_class_history (
    student_id INT,
    student_name VARCHAR(50),
    old_class_id INT,
    new_class_id INT,
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### ✅ Trigger

```sql
DELIMITER //

CREATE TRIGGER track_class_change
BEFORE UPDATE ON students
FOR EACH ROW
BEGIN
    IF OLD.class_id <> NEW.class_id THEN
        INSERT INTO student_class_history
        VALUES (OLD.student_id, OLD.name, OLD.class_id, NEW.class_id, NOW());
    END IF;
END //

DELIMITER ;
```

---

### 🧠 Why

* Detect change using `OLD` vs `NEW`

### 🧠 HinEnglish

"Update hone pe check kiya class change hua kya → agar haan toh log karo."
